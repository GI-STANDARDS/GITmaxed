Se você está contribuindo para um projeto existente ou estendendo-o, quase sempre é mais fácil continuar usando a licença desse projeto. Para encontrá-la, procure um arquivo chamado `LICENSE` ou `COPYING` e dê uma olhada no `README` do projeto. Se você não encontrar uma licença, [pergunte a quem o mantém](/no-permission/#for-users).

Dependendo da licença do projeto original, usar a mesma licença pode ser uma exigência, e não apenas a opção mais fácil. (Veja a condição "mesma licença" de [algumas licenças](/licenses/).)

Algumas comunidades têm fortes preferências por licenças específicas. Se você quiser participar de uma delas, será mais fácil usar a licença preferida, mesmo que esteja começando um projeto totalmente novo, sem dependências existentes. Alguns exemplos:

{: .bullets}

* [Apache](https://www.apache.org/licenses/) exige a [Apache License 2.0](/licenses/apache-2.0/)
* A [Cloud Native Computing Foundation](https://github.com/cncf/foundation/blob/main/charter.md#11-ip-policy) exige a [Apache License 2.0](/licenses/apache-2.0/)
* A [GNU](https://www.gnu.org/licenses/license-recommendations.html) recomenda a [GNU GPLv3](/licenses/gpl-3.0/) para a maioria dos programas
* Os [pacotes npm](https://libraries.io/search?platforms=npm) usam de forma esmagadora as licenças [MIT](/licenses/mit/) ou a muito semelhante [ISC](/licenses/isc)
* O [OpenBSD](https://www.openbsd.org/policy.html) prefere a [Licença ISC](/licenses/isc/)
* As crates [Rust](https://rust-lang.github.io/api-guidelines/necessities.html#crate-and-its-dependencies-have-a-permissive-license-c-permissive) são licenciadas de forma esmagadora sob [MIT](/licenses/mit/) [`OR`](https://spdx.github.io/spdx-spec/v2.3/SPDX-license-expressions/#d42-disjunctive-or-operator) [Apache License 2.0](/licenses/apache-2.0/)
* Os plugins e temas do [WordPress](https://wordpress.org/about/license/) devem ser [GNU GPLv2](/licenses/gpl-2.0/) (ou posterior)
* As extensões e templates do [Joomla](https://tm.joomla.org/joomla-license-faq.html#can-i-release-an-extension-under-a-non-gpl-license) devem ser [GNU GPLv2](/licenses/gpl-2.0/) para o código PHP.

As comunidades vêm em todas as formas e tamanhos, e mais de uma comunidade pode ser pertinente (por exemplo, leve em conta [sua empresa](https://opensource.guide/legal/#what-does-my-companys-legal-team-need-to-know) se você trabalha em uma). Os exemplos acima são *muito* bem estabelecidos. Se a comunidade para a qual você está criando um projeto não tem tradições de licenciamento gravadas em pedra, ou se você não vê seu projeto como parte de nenhuma comunidade específica, tudo bem: [faça sua própria escolha de uma licença de código aberto](/).
