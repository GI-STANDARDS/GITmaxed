GitHub 希望帮助开发者为他们的源代码选择一个开源许可证。

如果你已经心中有数，并且有偏好使用的许可证，那很好！我们无意改变你的想法。但如果你被众多的选择弄得不知所措，也许我们能帮上忙。

## 并不全面

本站并非开源许可证的完整目录。绝大多数项目很可能都能由[首页](/)上重点推荐的三个选项很好地满足——选择与你类似的项目所[偏好](/community/)的许可证、最受欢迎的宽松许可证（[MIT](/licenses/mit/)）、或最受欢迎的 copyleft 许可证（[GNU GPLv3](/licenses/gpl-3.0/)）。万一你有这些选项未能覆盖的特定需求，我们也重点介绍了[另外几个值得考虑的许可证](/licenses/)，并设有一个关于[非软件项目许可证](/non-software/)的页面。

请查阅我们的[附录](/appendix/)，那里有一张表格收录了 [choosealicense.com 仓库](https://github.com/github/choosealicense.com)中编目的所有许可证；并参阅下方的链接，那里有*更多*许可证——你**无需**了解它们也能为你的项目选到一个很好的许可证。

### 更多资源

{: .bullets}

* 开源促进会（OSI）的常见问题解答：[我应该选择哪种开源许可证来发布我的软件？](https://opensource.org/faq#which-license)
* 自由软件基金会（FSF）关于[如何选择许可证的建议](https://www.gnu.org/licenses/license-recommendations.html)

* [The Legal Side of Open Source](https://opensource.guide/legal/)，一份涵盖许可与相关问题的开源指南

## 帮助我们改进它

Choosealicense.com 不仅仅是在谈论开源；本站本身也是开源的。看到了你觉得可以做得更好的地方？欢迎在 GitHub 上[复刻本项目](https://github.com/github/choosealicense.com)并提交 pull request。我们欢迎你的改进。
