Lorsque vous créez une œuvre de l’esprit (ce qui inclut le code), celle-ci est par défaut soumise au droit d’auteur exclusif. À moins d’inclure une licence stipulant le contraire, personne d’autre ne peut copier, distribuer ou modifier votre travail sans risquer des retraits, des mises en demeure ou des poursuites. Une fois que l’œuvre compte d’autres contributeurs (chacun titulaire de droits d’auteur), « personne » commence à vous inclure.

Même en l’absence de fichier de licence, vous pouvez accorder certains droits lorsque vous publiez votre code source sur un site dont l’utilisation suppose l’acceptation de conditions de service. Par exemple, si vous publiez votre code source dans un dépôt public sur GitHub, vous avez accepté les [conditions de service](https://help.github.com/articles/github-terms-of-service), par lesquelles vous autorisez autrui à consulter et forker votre dépôt. D’autres personnes peuvent ne pas avoir besoin de votre permission si des [limitations et exceptions au droit d’auteur](https://en.wikipedia.org/wiki/Limitations_and_exceptions_to_copyright) s’appliquent à leur situation particulière. Ni les conditions d’un site ni les limitations du droit d’auteur propres à une juridiction ne suffisent pour le type de collaboration que les gens recherchent habituellement sur un hébergeur de code public, comme l’expérimentation, la modification et le partage que favorise une licence open source.

Vous n’avez rien à faire pour *ne pas* proposer de licence. Vous pouvez toutefois ajouter une mention de droit d’auteur et une déclaration indiquant que vous ne proposez aucune licence, à un endroit bien visible (par exemple le README de votre projet), afin que les [utilisateurs](#for-users) n’y voient pas un oubli. Si vous comptez accepter les contributions d’autrui à votre projet sans licence, vous pouvez envisager, avec votre juriste, d’ajouter un accord de contribution afin de conserver l’autorisation, au titre du droit d’auteur, des contributeurs, même si vous n’accordez pas la même chose.

Interdire l’utilisation de votre code n’est peut-être pas ce que vous entendez par « aucune licence ». Une [licence open source](/) autorise la réutilisation de votre code tout en conservant le droit d’auteur. Si votre objectif est de renoncer totalement aux restrictions du droit d’auteur, essayez plutôt une [dédicace au domaine public](/licenses/#unlicense).

## Pour les utilisateurs
{: #for-users }

Si vous trouvez un logiciel dépourvu de licence, cela signifie généralement que vous n’avez aucune permission de ses créateurs pour l’utiliser, le modifier ou le partager. Même si un hébergeur de code comme GitHub vous permet d’en consulter et d’en forker le code, cela n’implique pas que vous soyez autorisé à utiliser, modifier ou partager le logiciel à quelque fin que ce soit.

Vos options :

{: .bullets}

- **Demandez gentiment aux mainteneurs d’ajouter une licence.** Sauf indication contraire forte dans le logiciel, l’absence de licence est probablement un oubli. Si le logiciel est hébergé sur un site comme GitHub, ouvrez une issue demandant une licence et incluez un lien vers ce site. Si vous êtes audacieux et qu’il est assez évident quelle licence convient le mieux, ouvrez une pull request pour ajouter une licence – voir « suggérer cette licence » dans la barre latérale de la page de chaque licence de ce site (par exemple [MIT](/licenses/mit/#suggest-this-license)).
- **N’utilisez pas le logiciel.** Trouvez ou créez une alternative placée sous une licence open source.
- **Négociez une licence privée.** Faites appel à votre juriste.
